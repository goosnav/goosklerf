## GOOSKLERF II — GAME BIBLE SKELETON (RULEBOOK + MODS + MODES + CAMPAIGNS + ASSETS)

### FRONT MATTER

1. **Cover + Tagline + Logo Page** — The “anti-brainrot game” positioning, logo lockup, edition number. 
2. **Credits + Contact + Version History** — Who made it, where to find updates, change log by version.
3. **How To Use This Book** — Explain “Core Rules + Mods + Modes” system and how to build a ruleset. 
4. **Quick Start (2 pages)** — The Easy rules as a learn-in-5-min pamphlet with a single diagram of the table layout. 
5. **Table of Contents + Icons Key (1 page)** — A legend of every symbol used (entity/fort/item/consumable/atk/hp). 

---

## PART I — COMPONENTS + TABLE SETUP

6. **What’s in the Box** — Cards, dice, tokens, recommended accessories, what’s optional.
7. **Card Anatomy (Full-Page Graphics)** — Callouts for each card type (stats, text box, “Special”, border rarity). 
8. **Stat System (One Graphic)** — “Roll ≤ ATK to hit; HP tracked with dice; super cards use multiple dice” in one visual. 
9. **Zones + Table Layout** — Battlefield, fortress zones, shop area, discard/deck/hand with a clean diagram. 
10. **Tokens & Tracking** — What markers are used for HP, buffs, “occupied”, round counters, score.

**Assets to include in this part:**

* A single “table layout” illustration. 
* 1-page “card anatomy” for each type.
* Icon legend (rook/duck/bomb/sword, etc.). 

---

## PART II — CORE ENGINE (THE RULES THAT NEVER CHANGE)

11. **Core Concepts (1 page)** — “What an Entity is, what a Fortress is, what Items can do, when Items can be used.” 
12. **Victory Conditions (Core)** — Occupy-all-fortresses, last-man-standing, special scenario (plus optional alt endings). 
13. **Setup (Core)** — Card distribution options + 7-card hand + deck-browsing rule (no random draws during play). 
14. **Turn Structure (Core)** — Play (use 3 cards), Move, Attack, Draw (choose 3). 
15. **Combat Loop (Core)** — Attack phase, Defense phase, Victory resolution with the roll-under stat mechanic. 
16. **Damage + Defeat + Restoration (Core or Advanced Toggle)** — When things die, where they go, and whether HP resets after fights. 
17. **Special Text Priority** — How card text overrides rules and how to resolve conflicts.
18. **No Card Stealing (Default Rule)** — Items always go to the owner’s discard unless a specific mod says otherwise (and what that mod requires).

**Assets to include in this part:**

* “One full example turn” walkthrough with arrows.
* “One combat example” with 2–3 dice rolls shown visually.
* A one-page “Core Rules Reference” tear-out.

---

## PART III — RULE LEVELS (EASY / INTERMEDIATE / ADVANCED)

19. **Rule Level Selector Page** — A single page that says: Easy = Core only; Intermediate = Core + Shop; Advanced = everything.

### 19A. EASY (Core Only)

20. **Easy Rules (verbatim rules block)** — The minimum playable system with zero optional mechanics.

### 19B. INTERMEDIATE (Core + Shop)

21. **Shop Mod (Official)** — How the shop is created and how buying works (1 purchase for 2 cards at start of turn). 
22. **Shop Variants (Optional)** — Community shop vs personal shops, refill rules, faster drafting option. 

### 19C. ADVANCED (Core + Shop + Advanced Combat + Limits)

23. **Parry Mod** — Parrying rules, what cannot be parried. 
24. **Retreat Mod** — Retreat rules and “pot shot”/final chance. 
25. **Bonus Move on Death Mod** — The “final bonus move” clause and how it’s resolved. 
26. **Fortress Destruction Mod** — “If fortress destroyed, defenders die (after bonus move).” 
27. **Trait Caps + Battlefield Cap Mod** — Min/max tables and entity limit on battlefield. 
28. **Timed Turns Mod (Party Speed)** — “1 minute max per non-combat turn” and how to enforce it cleanly. 

**Assets to include in this part:**

* “Rule level comparison” table (what’s included).
* Tiny icons indicating required components (dice/tokens/shop cards).

30. **Economy Mods** — Shop variants, hand-size variants, draw variants (but keep “choose cards” as default).
31. **Combat Mods** — Parry, retreat, variable damage, boss dice rules.
32. **Objective Mods** — First-strike incentives, showdown rounds, alternative win conditions. 
33. **Party/Chaos Mods** — IRL challenge cards, consent-based “house rules”, “complexity is free” fun pack.
34. **Legacy Mods** — Inventory/capacity re-enabled, Defense stat enabled (for legacy integration).

**Assets to include in this part:**

* 1-page “Build-a-Ruleset” checklist (circle the mods you’re using).
* Print-and-cut mod cards (optional) so tables can toggle mods physically.

49. **Compatibility Toggles** — A “Legacy On/Off” mod checklist (Defense rules enabled, inventory enabled, etc.).
